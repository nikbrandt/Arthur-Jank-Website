Arthur: Privacy Policy

Any message sent via direct message to the Discord bot Arthur is logged publicly in Arthur's support server, if it is not a command. This is a feature built in good fun; users can have conversations with developers about whatever they please. In the five years Arthur has been running, only once has private information been logged -- in this case, it was removed and the original user was notified.

You shouldn't be sending private data to Discord bots regardless, but if something is sent and you'd like it removed, please contact Nikolas Brandt (Gymno#4741 on Discord, also available in the support server previously linked) for quick removal of the message(s).

Please note: although these direct messages are "logged" in the sense that they are retained in place and can be looked up, they are only logged to a channel (#message-logs in the support server) within Discord; nowhere else. As such, Arthur itself does not collect any personal user data. This means the messages may fall under Discord's privacy policy as well, but that should be a given since you're using Discord in the first place.

Beyond this, Arthur does not collect any personal data. Basic command usage logs are kept and a database is retained to build basic functionality (toggling server/user settings, handling leveling, keeping polls alive after restarts, etc.), but none of this functionality uses personally identifiable information: only Discord snowflake IDs (strings of numbers that represent users, servers, messages, etc.) are stored. Message content, usernames, and more may be temporarily stored in RAM while Arthur is processing messages, but will never be retained.

Further, echoing a friend's privacy policy, Arthur is a passion project of mine with my only intent in mind being "create something cool!". I have no motivation to do anything malicious with your data, and I'm happy to have a conversation if you're concerned. Additionally, all of Arthur's code is open source, which should provide absolute transparency and assurance that I'm not doing anything with your data.

If you're curious, the old TOS, which was more of the privacy policy, explains in detail what data is stored. Again, none of it is personally identifiable.
